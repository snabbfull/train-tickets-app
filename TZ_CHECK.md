## ⚠️ Рекомендации

1. **Параметры поиска** — фильтры (класс вагона, цена, время) применяются на клиенте. API routes поддерживает эти параметры — можно передавать их в запрос для оптимизации.

2. **Дата обратно** — в SearchForm используется `date_start_arrival`; в API есть `date_end` для даты обратно. Проверить соответствие логики.

3. **Бельё включено** — в seats API есть `is_linens_included`; при `true` стоимость белья не должна прибавляться. Проверить реализацию в seatsSectionUtils / order.
